# Contributing to Promps

Thank you for your interest in contributing to Promps!

---

## ⚠️ Tech Preview Status

**Promps v0.0.1 is a Technology Preview** - an early release to test the concept and gather feedback from the community.

**What this means for contributors:**
- Focus is on **testing and feedback** rather than code contributions
- Core features are minimal but functional
- Your input will directly influence future development
- Bugs and rough edges are expected

---

## [Test] We're Looking for Testers!

**Promps is in Technology Preview (v0.0.1)** and we're actively seeking testers to help validate the concept and improve the project.

### How to Help as a Tester

1. **Try the application**
   - Download and install Promps
   - Create prompts using the visual block interface
   - Test different block combinations

2. **Report bugs**
   - Use the [Bug Report template](.github/ISSUE_TEMPLATE/bug_report.md)
   - Include steps to reproduce, expected vs actual behavior
   - Attach screenshots if possible

3. **Share feedback**
   - What works well?
   - What's confusing or difficult?
   - What features would you like to see?

4. **Test on different platforms**
   - Linux, Windows, macOS
   - Report platform-specific issues

---

## [Bug] Reporting Bugs

**Before submitting:**
- Check [existing issues](https://github.com/BonoJovi/Promps/issues) to avoid duplicates
- Make sure you're using the latest version

**When reporting:**
1. Use the Bug Report template
2. Describe:
   - Steps to reproduce
   - Expected behavior
   - Actual behavior
3. Include:
   - OS and version
   - Promps version
   - Screenshots/logs (if applicable)

---

## [Idea] Suggesting Features

We welcome feature suggestions! Please:

1. Use the [Feature Request template](.github/ISSUE_TEMPLATE/feature_request.md)
2. Describe:
   - What you want to accomplish
   - Why this feature would be useful
   - How you envision it working
3. Check if similar requests already exist

---

## [Fix] Contributing Code (Future)

**Note:** For v0.0.1, we're focusing on testing and feedback. Code contributions will be welcomed in future versions.

When code contributions are accepted:
1. Fork the repository
2. Create a feature branch
3. Write clear commit messages
4. Add/update tests
5. Follow existing code style
6. Submit a pull request

### Code Style
- **Rust**: Use `cargo fmt` and `cargo clippy`
- **JavaScript**: Follow existing patterns
- Keep code simple and readable

---

## [Note] Documentation

Documentation improvements are always welcome:
- Fix typos or unclear explanations
- Add examples
- Translate to other languages

---

All contributors must follow our [Code of Conduct](CODE_OF_CONDUCT.md).

# Promps への貢献

Promps への貢献に興味を持っていただき、ありがとうございます！

---

## ⚠️ テックプレビュー状態

**Promps v0.0.1 はテクノロジープレビュー** です - コンセプトをテストし、コミュニティからのフィードバックを収集するための早期リリースです。

**これが貢献者にとって意味すること:**
- コード貢献よりも**テストとフィードバック**に焦点
- コア機能は最小限ですが機能的
- あなたの意見が将来の開発に直接影響します
- バグや粗削りな部分は想定内です

---

## [Test] テスターを募集しています！

**Promps はテクノロジープレビュー（v0.0.1）** であり、コンセプト検証とプロジェクト改善のためのテスターを積極的に募集しています。

### テスターとして協力する方法

1. **アプリケーションを試す**
   - Promps をダウンロードしてインストール
   - ビジュアルブロックインターフェースでプロンプトを作成
   - 異なるブロックの組み合わせをテスト

2. **バグを報告**
   - [バグレポートテンプレート](.github/ISSUE_TEMPLATE/バグレポート.md) を使用
   - 再現手順、期待される動作、実際の動作を含める
   - 可能であればスクリーンショットを添付

3. **フィードバックを共有**
   - うまく機能している点は？
   - 分かりにくい点や難しい点は？
   - どんな機能が欲しいですか？

4. **異なるプラットフォームでテスト**
   - Linux、Windows、macOS
   - プラットフォーム固有の問題を報告

---

## [Bug] バグの報告

**報告前の確認：**
- [既存の Issue](https://github.com/BonoJovi/Promps/issues) で重複を確認
- 最新バージョンを使用していることを確認

**報告内容：**
1. バグレポートテンプレートを使用
2. 説明する項目:
   - 再現手順
   - 期待される動作
   - 実際の動作
3. 含める情報:
   - OS とバージョン
   - Promps バージョン
   - スクリーンショット/ログ（該当する場合）

---

## [Idea] 機能の提案

機能提案を歓迎します！以下をお願いします：

1. [機能リクエストテンプレート](.github/ISSUE_TEMPLATE/機能リクエスト.md) を使用
2. 説明する項目:
   - 実現したいこと
   - なぜこの機能が有用か
   - どのように動作することを想定しているか
3. 類似のリクエストが既に存在しないか確認

---

## [Fix] コードの貢献（将来）

**注意:** v0.0.1 では、テストとフィードバックに焦点を当てています。コード貢献は将来のバージョンで歓迎します。

コード貢献が受け入れられる際：
1. リポジトリをフォーク
2. 機能ブランチを作成
3. 明確なコミットメッセージを記述
4. テストを追加/更新
5. 既存のコードスタイルに従う
6. プルリクエストを提出

### コードスタイル
- **Rust**: `cargo fmt` と `cargo clippy` を使用
- **JavaScript**: 既存のパターンに従う
- コードをシンプルで読みやすく保つ
